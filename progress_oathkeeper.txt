## Iteration 2 - US-002: System Behavior Description Exclusion
- Added exclusion patterns to Detector for non-agent subjects (system descriptions, user instructions)
- Added IsSystemDescription() public method for direct classification
- Integrated exclusion check into DetectCommitment() as early return before pattern matching
- Exclusion patterns cover: "the X will", "X.sh will", "it/this/that will", "you can/should/will"
- Added 3 new test functions: TestExcludeSystemBehaviorDescriptions, TestIsSystemDescription, TestAgentCommitmentsStillDetected
- Files changed: pkg/detector/detector.go, pkg/detector/detector_test.go
- Learnings for future iterations:
  - Exclusion-first approach (check exclusions before matches) is cleaner than trying to make patterns more specific
  - Data-driven regex exclusions are easy to extend without changing logic
  - Current patterns use ^ anchor for start-of-string matching on exclusions — works well for sentence-level analysis
  - Detector struct now has both `patterns` and `exclusions` slices
