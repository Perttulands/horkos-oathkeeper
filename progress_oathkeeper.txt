## Iteration 6 - US-006: 30-Second Grace Period
- Implemented pkg/grace package with GracePeriod scheduler
- GracePeriod uses time.AfterFunc to delay verification by configurable duration (default 30s)
- Schedule() accepts commitmentID, detectedAt, and callback — runs verify function after delay
- Cancel() stops a pending verification by ID (returns true if was pending)
- Stop() cancels all pending verifications (for clean shutdown)
- Pending() returns count of commitments awaiting verification
- VerifyFunc signature decouples grace period from verifier implementation
- 11 test cases: scheduled verification, wait timing, not-backed, multiple commitments, detectedAt passthrough, cancel, cancel-nonexistent, stop, pending count, default duration, outcome fields
- Files changed: pkg/grace/grace.go, pkg/grace/grace_test.go
- Learnings for future iterations:
  - time.AfterFunc is the idiomatic Go approach for deferred execution
  - cancelCh channel + select for cancellation is cleaner than context.Context for simple cases
  - Copy-map-then-clear pattern in Stop() avoids holding lock during cleanup
  - VerifyFunc as function type keeps the grace package decoupled from verifier package

**Summary:**

## Iteration 9 - US-009: Periodic Re-Check of Unresolved Commitments
- Implemented pkg/recheck package with Rechecker that periodically re-checks commitments
- RunOnce() performs a single cycle: fetch → for each commitment: check expiration → verify → alert/update
- Start()/Stop() manage a background ticker goroutine using channel + sync.Once for safe shutdown
- Function type dependencies (FetchFunc, VerifyFunc, UpdateFunc, AlertFunc) decouple from storage/verifier/alerts
- Config struct with defaults (5min interval, 3 max alerts)
- Expiration checked before verification to avoid unnecessary work
- Max alert cap prevents notification fatigue (alert_count >= maxAlerts skips alert)
- Verify errors treated as unbacked (graceful degradation, consistent with verifier pattern)
- 12 test cases: backed, unbacked+alert, expired, max-alerts-skip, multiple commitments, last_checked, periodic running, clean stop, fetch error, verify error continues, expiration-before-verify, default interval
- Files changed: pkg/recheck/recheck.go, pkg/recheck/recheck_test.go
- Learnings for future iterations:
  - Function type pattern (from grace package) continues to scale cleanly for decoupling
  - sync.Once for Stop() prevents double-close panic on channel
  - Expiration-first check is an optimization — avoids calling verify for already-expired commitments
  - Status constants defined locally; will move to shared package when storage is built
