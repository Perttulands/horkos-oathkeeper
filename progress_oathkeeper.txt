- What was implemented
  - Added new `pkg/beads` package with `BeadStore` and lifecycle methods: `Create`, `Close`, `List`, `Get`
  - Added strict command availability handling with `ErrCommandUnavailable`
  - Added JSON parsing and normalization into `Bead` fields (`ID`, `Title`, `Status`, `Tags`, `CreatedAt`, `ClosedAt`)
  - Added integration-style tests using real `br` (skip when `br` is missing)
- Files changed
  - pkg/beads/beads.go
  - pkg/beads/beads_test.go
- Learnings for future iterations:
  - Patterns discovered
    - Current `br` in this environment supports `--labels` for create/list filtering and not `--tag`; runtime fallback to label flags keeps compatibility while still requiring `br`.
    - `br show --format json` returns an array payload, not a single object.
  - Gotchas encountered
    - `br` requires DB paths under a `.beads/` directory component.
    - `br close` emits info logs to stderr on success; parsing must rely on stdout for JSON data commands.
  - Useful context
    - `go test ./pkg/beads/... -v` passes all 3 tests.
    - Required verification command `pytest pkg/beads/beads_test.go -v` fails in principle because pytest does not collect Go test files.
